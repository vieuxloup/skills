# Book Analysis Patterns

Templates for analyzing different book genres when turning them into skills.

---

## Pattern A: 方法论/框架类 (e.g., 《金字塔原理》《高效能人士的七个习惯》)

**Prompt:**

```
这是《书名》全文。通读后，按以下结构输出：

一、核心框架
- 中心论点是什么？
- 框架的层级关系（用树状或流程图表示）
- 各模块之间的依赖/递进关系

二、场景拆解
按书中实际场景划分（如表达、思考、解决问题、演示），每个场景：
- 场景名称
- 该场景的完整逻辑流程
- 每步对应的书中原始方法/原则
- 检查清单（书中是否有对应的checklist）
```

**Output structure:**

```
# 书名 Skill

## 场景1: [场景名]
### 适用场景
### 逻辑流程
### 方法清单
### 检查清单

## 场景2: [场景名]
...
```

## Pattern B: 职场/沟通类 (e.g., 《非暴力沟通》《关键对话》《影响力》)

**Prompt:**

```
这是《书名》全文。通读后，按以下结构输出：

一、核心理念
- 本书最核心的一个原则/理念是什么？
- 围绕它的几个支柱性方法

二、对话/沟通阶段拆解
按实际沟通的推进阶段拆分场景，每个场景：
- 触发条件（什么情况下使用）
- 该阶段的操作步骤
- 具体话术/技巧（保留书中原始描述）
- 常见错误及书中给出的纠正方法

三、冲突处理流程
- 书中是否给出了冲突升级时的处理路径？
- 如果涉及多个参与方，各方的视角是什么？
```

## Pattern C: 认知/思维类 (e.g., 《认知天性》《思考，快与慢》《系统思考》)

**Prompt:**

```
这是《书名》全文。通读后，按以下结构输出：

一、核心认知模型
- 本书揭示/构建了什么模型？
- 模型的核心概念（保留原始术语）
- 模型如何解释或预测行为/现象

二、应用场景
- 什么条件下可以用这个模型分析问题？
- 该模型与日常决策的映射关系
- 如何验证这个模型是否适用当前场景

三、实用工具
- 书中是否给出了思维工具/决策框架？
- 每件工具的使用条件、步骤、注意事项
- 常见误用场景
```

## Pattern D: 原则/哲学类 (e.g., 《原则》《孙子兵法》《毛选》中的方法论篇章)

**Prompt:**

```
这是《书名》全文。通读后，按以下结构输出：

一、原则体系
- 列出所有核心原则（保留原文表述，不概括）
- 原则之间的优先级/冲突处理逻辑
- 应用原则时的前置条件

二、场景适配
- 每个原则最适合的应用场景
- 哪个场景调用哪个原则（建立映射表）
- 多原则冲突时的决策优先级

三、转化指引
- 原则如何转化为具体行动？
- 不同层级（战略/战术/执行）分别对应哪些原则
- 执行后的检查/复盘方法
```
