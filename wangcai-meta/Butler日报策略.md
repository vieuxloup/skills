# 旺财Butler日报执行策略

> 当收到定时任务（⚠️ EXECUTION CRON）时，按此策略执行。

## 优先级策略

```
时间充足(≤120s) → 正常生成本日报(PDF) → 推送 → 写日志
可能超时(>120s) → 先出精简版(text) → 推送 → 继续生成PDF → 后补推送
严重不足(<30s)  → 直接推送关键摘要(text) → 写日志
```

## 具体操作

### 正常模式（标准时长）
1. 搜集信息
2. 整理内容
3. 生成PDF
4. 推送到微信
5. 写exec-log日志

### 降级模式（预计超时）
当发现内容太多/搜索太慢可能超时时：

1. **先发精简版** — 纯文字，不转PDF，3-5条要点直接推微信
2. **告知老板** — "今日日报精简版先到，详版PDF稍后补送"
3. **继续生成** — 后台继续做PDF（如果系统允许）
4. **补推** — PDF生成后单独推送
5. **写日志** — 记录超时原因，为下次优化做准备

### 紧急模式（时间严重不足）
1. 只推送最重要的1-2条关键信息
2. 写日志说明情况
3. 在W5反思中分析超时根因

## 每条日报的已知耗时参考

| 日报名称 | 平均耗时 | 策略 |
|---------|:--------:|------|
| 无人驾驶专用车行业日报(7:30) | ~150s | 正常模式，接近阈值注意降级 |
| 每日早报-财经+社会(8:00) | ~300s+ ❌ | 已连续失败6次 → 优先精简版 |
| 雷击防护产业日报(8:30) | ~600s ❌ | 严重超时 → 必须降级 |
| 充电基础设施政策内参(9:00) | ~300s ❌ | 连续超时 → 精简版优先 |
| 每周社媒热点报告(周一9:00) | ~300s ❌ | 精简版优先 |
| 出租寻租信息监控(16:00) | ~1448s | 时长正常但内容量极大 |
