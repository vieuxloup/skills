# SKILL 01: 古籍小学 - 校勘、音韵、训诂、注释总览

> **小学**：中国传统语言文字学的总称，汉代以来指**文字**、**音韵**、**训诂**三学，清代发展为更广义的**文献整理学**，包括校勘、音韵、训诂、注释等基础工序，是理解和研究古代文献的基石。

---

## 一、小学的内涵与项目定位

### 1.1 传统小学的四大支柱

| 领域 | 核心内容 | 代表著作 | 项目对应工序 |
|-----|---------|---------|------------|
| **校勘** | 版本对比、文字考订、异文鉴别 | 《尚书》孔安国传、段玉裁《经韵楼集》 | 多版本互校、标注完整性维护、表格校勘 |
| **音韵** | 古音构拟、反切注音、今音标注 | 《切韵》《广韵》《音学五书》 | 正音标注、拼音标注、特殊读音词库 |
| **训诂** | 词义解释、引申义辨析、通假字考证 | 《说文解字》《尔雅》《经传释词》 | （待实现）词义标注、通假字标注 |
| **注释** | 典故考证、背景阐释、名物制度 | 三家注（集解/索隐/正义）、《史记会注考证》 | 三家注标注、人物/地点/官职实体标注 |

### 1.2 项目的小学工序范围

本项目是**《史记》知识库**的构建工程，小学工序服务于**结构化知识抽取**的最终目标：

```
【数据流向】
原始底本（简体分章）
  ↓
① 句读工序：建立文本结构骨架（最前置）
  └─ 句读与标点校勘（SKILL_01f）- **断句、标点规范、质量控制**
  ↓
② 校勘工序：建立可靠定本
  ├─ 多版本互校（SKILL_01b）
  ├─ 表格校勘规范（SKILL_01c）
  └─ 繁简体处理（SKILL_01e）- **异体字统一、繁简转换**
  ↓
③ 音韵工序：标注读音
  ├─ 正音标注（SKILL_01d）
  └─ 拼音生成（SKILL_01d）
  ↓
④ 结构分析：语义标注
  ├─ 章节切分（SKILL_02a）
  ├─ 区块与韵文（SKILL_02b）
  ├─ 三家注标注（SKILL_02c）- **注释工序的体现**
  ├─ 结构语义（SKILL_02d）
  └─ 词法分析（SKILL_02e）
  ↓
⑤ 实体标注：知识抽取
  ├─ 人物标注（SKILL_03a）
  ├─ 地点标注（SKILL_03a）
  ├─ 时间标注（SKILL_03a）
  ├─ 官职标注（SKILL_03a）
  └─ 书名标注（SKILL_03a）
  ↓
⑥ 质量验证：确保标注质量（所有标注完成后）
  └─ 标注完整性维护（SKILL_01a）- **验证标注未篡改原文**
  ↓
⑦ 知识图谱生成：最终产品
  ├─ 实体关系网络
  ├─ 事件时间线
  └─ HTML交互展示
```

**与传统小学的差异**：
- **训诂工序**：尚未全面实施（三家注中部分体现）
- **注释工序**：主要通过三家注标注体现，尚未覆盖典故、制度等深层注释
- **繁简处理**：传统小学无此工序，为现代数字化需求新增（SKILL_01e），归入校勘工序

## 二、句读工序（部分实现）

### 2.1 句读的目标

**核心目标**：建立准确的文本结构骨架，为所有后续工序奠定基础

**为何句读在最前面**：
1. **句读是文本的"骨架"**：错误的句读会导致语义切分错误
2. **影响所有下游工序**：实体边界、引文识别、韵文分析都依赖正确的句读
3. **标点是原文的一部分**：标点符号规范影响完整性检查的规则

### 2.2 句读工序分解

| 子工序 | 文档 | 核心内容 | 实施状态 |
|-------|------|---------|---------|
| **句读与标点校勘** | [SKILL_01f](SKILL_01f_句读和标点校勘.md) | 无标点古籍的AI自动断句、已标点版本的标点反思、标点符号规范、完整性检查中的标点处理 | ⚠️ 部分实现（规范已定，工具待开发） |
| **OCR影印古籍句读排版** | [SKILL_01i](SKILL_01i_OCR影印古籍句读排版.md) | 从"无标点+OCR噪声+影印排版"三重困境的生文本，流水线式产出Markdown可读底本 | ⚠️ 方法论已定（案例：《读史记十表》） |

### 2.3 句读工作流程

**三种场景**：

1. **无标点古籍**：
   - 使用 gj.cool API 或本地模型自动断句
   - 人工校对（重点检查长句、引文、韵文）
   - AI反思验证
   - 生成校对报告
   - **特殊子场景：OCR影印古籍**（同时含OCR噪声与排版崩坏），见 [SKILL_01i](SKILL_01i_OCR影印古籍句读排版.md)

2. **已标点版本**：
   - 提取标点模式
   - 多版本对照（中华书局/维基/四库）
   - 标记可疑句读
   - AI反思建议
   - 人工决策修正

3. **完整性检查**：
   - 标点视为"结构性添加"（原文无标点时）
   - 规范化映射消除全角/半角差异
   - 引号差异通过规范化忽略

### 2.4 标点符号规范

**基本原则**：
- ✅ **全角优先**：所有标点必须使用全角字符
- ✅ **语义优先**：标点服从语义，不拘泥于现代语法
- ✅ **从俗从简**：常用标点优先

**引号嵌套规则**（从外到内）：
```
第一层：" "  （弯双引号）
第二层：' '  （弯单引号）
第三层：" "  （重复第一层）
```

**禁止事项**：
- ❌ 半角标点（`,` `.` `?` `!` `"` `'` 等）
- ❌ 直角引号（`「」` `『』`，日文用法）
- ❌ 引号嵌套错误（第一层用单引号）

## 三、校勘工序（已实现）

### 3.1 校勘的目标

**核心目标**：建立准确、一致、可溯源的定本（标准底本）

**三大原则**：
1. **标注铁律**：标注只添加`〖TYPE〗`符号，不修改原文字符（增删改）
2. **版本溯源**：所有文字改动必须有版本依据
3. **完整性验证**：去除标注符号后必须与原始txt逐字相同

### 3.2 校勘工序分解

| 子工序 | 文档 | 核心内容 | 实施状态 |
|-------|------|---------|---------|
| **多版本互校** | [SKILL_01b](SKILL_01b_多版本互校底本.md) | 对照简体底本、维基繁体、四库版等多版本，记录异文，确定定本 | ✅ 已实现 |
| **表格校勘** | [SKILL_01c](SKILL_01c_表格校勘规范.md) | 年表、世系表等表格结构的校勘规范，确保表格语义正确 | ✅ 已实现 |
| **繁简体处理** | [SKILL_01e](SKILL_01e_繁简体处理.md) | 异体字统一、繁简转换、字形规范化，确保文本字形一致性 | ✅ 已实现 |

### 2.3 校勘来源

**可用版本资源**（按优先级）：
1. **简体底本** - `corpus/archive/chapter/*.txt`（130篇分章，主校本）
2. **维基正文** - `corpus/shiji/wikisource_shiji/*.html`（繁体，超链接标注）
3. **四库版** - `corpus/shiji/史记四库.txt`（简体，四库全书版）
4. **繁体正文** - `corpus/shiji/史記正文.繁体.txt`（纯文本繁体）
5. **三家注本** - `corpus/shiji/wikisource_sanjia/*.html`（含集解/索隐/正义，有校勘标记）

**校勘工具**：
- Python脚本：`scripts/lint_text_integrity.py`（完整性验证）
- 手工对校：复杂异文、表格结构需人工判断

### 2.4 常见校勘问题类型

| 错误类型 | 典型示例 | 解决方案 |
|---------|---------|---------|
| **衍文** | 多出的重复字词 | 对照多版本删除 |
| **脱文** | 缺漏的字词 | 对照多版本补全 |
| **讹字** | 形近字误写（"已"误为"己"） | 根据上下文和版本对校 |
| **倒文** | 字序颠倒（"国定于"误为"于定国"） | 恢复正确语序 |
| **标点错误** | 句读不当、引号缺失 | ⚠️ **只允许补充标点，禁止改变标点类型**（全角/半角、引号样式等） |
| **表格错位** | 年代与事件对应错误 | 参照SKILL_01c规范修复 |
| **网络爬取垃圾** | 整章被替换为网站索引 | 从权威版本重新提取 |

### 3.3 工序位置

```
corpus/archive/chapter/*.txt        底本（网络来源，可能有错）
        ↓ 句读与标点（SKILL_01f）← 最前置，建立文本骨架
        ↓ 多版本互校（SKILL_01b）
        ↓ 表格校勘（SKILL_01c）
        ↓ 繁简体处理（SKILL_01e）← 异体字统一、字形规范化
curation_base/*.txt         定本（多版本互校后）
        ↓ 同步到
corpus/archive/chapter/*.txt        定本（更新后的底本）
        ↓ 切分+编号（SKILL_02a）
chapter_md/*.tagged.md       标注文本
        ↓ 实体标注（SKILL_03a）
        ↓ 完整性维护（SKILL_01a）← 所有标注完成后验证
        ↓ 事件提取 ...
```

**核心原则**：
- **句读先行**：标点确定后，才能进行多版本对照和文本校勘
- **校勘只改原文txt**：不改标注文件，标注文件应忠实反映定本
- **完整性验证在最后**：所有标注工作完成后，验证标注未篡改原文

## 四、音韵工序（部分实现）

### 4.1 音韵的目标

**核心目标**：为《史记》全文标注准确读音，支持：
- **正字音**：多音字、通假字、古今异读的正确读音标注
- **拼音显示**：HTML页面中为每个汉字标注拼音（包括声调）
- **特殊读音**：人名、地名等专有名词的特殊读音处理

### 4.2 音韵工序分解

| 子工序 | 文档 | 核心内容 | 实施状态 |
|-------|------|---------|---------|
| **正音标注** | [SKILL_01d](SKILL_01d_正音与拼音标注.md) | 构建特殊读音词库（如"单于"dān yú），使用pypinyin库生成拼音 | ✅ 已实现 |
| **拼音显示** | [SKILL_01d](SKILL_01d_正音与拼音标注.md) | HTML页面中为汉字添加`<ruby>`标签显示拼音注音 | ✅ 已实现 |

### 4.3 音韵工具链

**Python工具**：
- `pypinyin`库：汉字转拼音（支持多音字词组判断）
- 自定义词库：`docs/data/pinyin-custom-dict.txt`（1,500+特殊读音）

**前端工具**：
- `simp-trad-converter.js`：繁简动态切换
- `OpenCC.js`：简体→繁体算法转换
- `s2t-custom-variants.json`：36条自定义纠错词表

### 4.4 音韵的特殊性

**与现代拼音的差异**：
- **古音构拟**：尚未实现（需参考《广韵》等韵书）
- **反切注音**：三家注中保留原文（如"单，音禅"），不转为拼音
- **入声字标记**：尚未实现

**与校勘的关系**：
- **繁简体处理**：虽然涉及字形转换，但本质是校勘问题（异体字统一），已归入校勘工序（SKILL_01e）
- **音韵标注依赖字形**：拼音标注需要在繁简体统一后进行，确保字形一致

## 五、训诂工序（待实现）

### 5.1 训诂的目标

**核心目标**：解释古汉语词义、辨析引申义、标注通假字

**训诂的三大功能**：
1. **释义**：解释词语的本义和引申义（如"走"本义为"跑"）
2. **通假**：标注通假字及其本字（如"说"通"悦"）
3. **词类活用**：标注使动、意动、名词作状语等用法

### 5.2 训诂在项目中的体现

**已有部分实现**：
- **三家注释义**：集解/索隐/正义中包含大量词义解释（如"某，某也"）
- **实体标注**：人名、地名、官职标注中隐含了词义识别

**尚未实现**：
- **系统性词义标注**：逐词标注词性、本义、引申义
- **通假字标记**：尚无专门的通假字标注语法
- **词类活用标记**：尚无专门的语法标注

### 5.3 训诂的实施规划

**可能的实施方案**（待讨论）：
1. **词义标注语法**：设计`〖训词义〗`标记（如`〖训走=跑〗`）
2. **通假字语法**：设计`〖通本字〗`标记（如`〖通说=悦〗`）
3. **词典关联**：链接到《说文解字》《汉语大词典》等数字词典
4. **LLM辅助标注**：使用LLM批量识别通假字和词类活用

**优先级**：低（P2，当前聚焦实体标注和知识图谱构建）

## 六、注释工序（部分实现）

### 6.1 注释的目标

**核心目标**：阐释历史背景、考证典故、解释名物制度

**注释的四大类型**：
1. **人物注**：人物生平、家世、事迹考证
2. **地理注**：古今地名对应、地理方位考证
3. **典故注**：引用的历史事件、成语典故来源
4. **名物注**：古代器物、制度、礼仪的解释

### 6.2 注释在项目中的体现

**已实现**：
- **三家注标注**：[SKILL_02c](SKILL_02c_三家注标注.md) - 结构化标注集解/索隐/正义
- **实体标注**：[SKILL_03a](SKILL_03a_实体标注.md) - 人名、地名、官职、书名等实体标注
- **书名引用**：使用`〖^书名〗`标记注释中引用的文献

**部分实现**：
- **人物关系**：通过实体链接部分体现（如"其父某"）
- **地理考证**：三家注中《正义》大量地理考证已标注

**尚未实现**：
- **典故标注**：尚无专门的典故标注语法（如"鸿门宴"的典故来源）
- **制度名物注**：尚无专门的制度名物标注（如"封禅"的礼仪解释）
- **修辞标注**：尚无修辞格标注（如互文、借代等）

### 6.3 三家注的特殊性

**三家注的内容分类**（详见[SKILL_02c](SKILL_02c_三家注标注.md)）：

| 注家 | 朝代 | 主要内容 |
|-----|------|---------|
| **集解**（裴骃） | 南朝宋（5世纪） | 训诂字义、人物地名考证、汇集前人注解、校勘异文 |
| **索隐**（司马贞） | 唐（8世纪） | 音韵注音（反切/直音）、文字辨析、补正集解、补录佚文 |
| **正义**（张守节） | 唐（8世纪） | 地理考证（引《括地志》）、历史背景阐释、较长解说 |

**标注策略差异**：
- **音韵注**（如"音X"、"X反"）：不标注实体，保留原文
- **释义注**（如"某，某也"）：标准实体标注
- **引书注**：书名用`〖^书名〗`标注
- **校勘记**：保留纯文本，不标注

### 6.4 注释的实施规划

**待实现的注释功能**：
1. **典故标注**（2周）
   - 设计`〖典典故名〗`语法
   - 构建《史记》典故索引（基于"太史公曰"、典故成语等）
   - 链接到典故出处（章节编号）

2. **制度名物标注**（2周）
   - 设计`〖制制度名〗`语法
   - 构建古代制度词表（封禅、朝觐、封爵等）
   - 链接到《正义》中的制度解释

3. **修辞标注**（1周）
   - 设计修辞格标记（互文、借代、用典等）
   - 标注《史记》中的经典修辞段落

**优先级**：中（P1，典故标注对理解文本有重要价值）

### 7.2 关键依赖说明

**P0级依赖**：
- **句读与标点**是最前置工序（错误的句读会导致所有下游工序失效）
- **章节切分**是实体标注的前提（需要段落编号定位）
- **标注完整性**是最终验证工序（所有标注完成后，验证标注未篡改原文）

**相互增强**：
- **句读标点** + **实体标注** → 引文边界准确，人名地名不会跨句断开
- **正音标注** + **实体标注** → 人名地名的准确读音
- **三家注标注** + **实体标注** → 注释中的人物地名链接
- **繁简体处理** + **拼音标注** → 繁体版的拼音也能正确显示

### 10.1 质量检查清单

**句读质量**：
- [ ] 断句准确率 >95%（人工校对后）

- [ ] 无嵌套标注错误（`--check-nested`）
- [ ] 表格结构语义正确（参照SKILL_01c规范）
- [ ] 异文记录完整（有版本依据）

**音韵质量**：
- [ ] 多音字读音正确（特殊读音词库覆盖）
- [ ] 拼音声调准确（pypinyin库正确调用）
- [ ] 繁体转换准确率 >97%（OpenCC + 自定义词表）

**注释质量**：
- [ ] 三家注类型标注正确（释义/音韵/引书/地理/校勘）
- [ ] 实体链接有效（人名/地名/官职/书名）
- [ ] 引书格式统一（`〖^书名〗`）

### 10.2 持续改进机制

**错误发现与修复流程**：
1. **读者反馈**：通过GitHub Issues提交错误报告
2. **自动检测**：定期运行完整性验证脚本
3. **版本对照**：发现异文时对照多版本确认
4. **批量修复**：使用Python脚本批量修复同类错误

**词库更新机制**：
- **拼音词库**：发现特殊读音时添加到`pinyin-custom-dict.txt`
- **繁简词表**：发现OpenCC错误时添加到`s2t-custom-variants.json`
- **v4.0规划**：使用LLM逐章分析，穷尽所有繁简转换错误（详见SKILL_01e）
